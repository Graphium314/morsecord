# Botの導入と運用

このページは、自分の環境でMorseCordを起動するBot管理者・開発者向けです。

## 必要なもの

- Rust 2021 EditionをビルドできるRustツールチェーン
- Discord Botトークン
- Botが接続できるDiscordサーバーとボイスチャンネル

Discord Developer PortalでBotの **Message Content Intent** を有効にしてください。コードはこのIntentを要求し、通常メッセージを読み取ってCWへ変換します。

Botをサーバーへ追加するときは、OAuth2 URL Generatorで `bot` と `applications.commands` の両スコープを選択します。

Botには、利用するチャンネルに応じて少なくとも次の権限が必要です。

- チャンネルを見る
- メッセージを送信する
- メッセージ履歴を読む
- リアクションを追加する
- ボイスチャンネルへ接続する
- ボイスチャンネルで発言する

## 設定ファイル

リポジトリ直下に `config.json` を作成します。

```json
{
  "token": "YOUR_DISCORD_BOT_TOKEN"
}
```

トークンをGitへコミットしないでください。`config.json` はこのリポジトリの `.gitignore` に含まれています。

## 起動

カレントディレクトリをリポジトリ直下にして実行します。

```bash
cargo run --release
```

起動時に以下が自動で行われます。

- `db.sqlite3` を作成または接続
- ユーザー別の速度・音程を保存する `cw_speed` テーブルを作成
- グローバルスラッシュコマンドを登録
- アクティビティを `Listening to 7.074MHz` に設定

`db.sqlite3` も `.gitignore` に含まれています。設定を維持する場合はバックアップ対象にしてください。

グローバルコマンドの追加・変更がDiscordクライアントに反映されるまで時間がかかる場合があります。

## 独自レッスン問題

`lesson_txt/` にUTF-8のテキストファイルを配置します。1行が1問です。

```text
CQ
599 BK
JA1ABC
```

利用者は次のように指定します。

```text
/cw-start-lesson probset:file:questions.txt
```

注意事項は[聞き取りレッスンの「独自問題ファイル」](聞き取りレッスン.md#独自問題ファイル)を参照してください。

## 終了

プロセスへCtrl+Cを送ると終了します。現在の通話・レッスン状態はメモリ上だけに保持されるため、再起動後は `/cw-join` からやり直してください。速度・音程の個人設定はSQLiteに保存されるため再起動後も残ります。

## 動作確認

テストを実行します。

```bash
cargo test
```

Discordへ接続せず、CW音声をWAVに書き出す補助バイナリも利用できます。

```bash
cargo run --release --bin cw_audio_test -- "CQ DE JA1ABC" output.wav 20 600
```

引数は順に、本文、出力先、WPM（省略時20）、周波数Hz（省略時600）です。

## トラブルシュート

### メッセージを投稿しても再生されない

- Developer PortalでMessage Content Intentが有効か確認します。
- Botにチャンネルの閲覧・ボイス接続・発言権限があるか確認します。
- 投稿先が `/cw-join` を実行したチャンネルか確認します。
- レッスン中でないか、メッセージ先頭が `;` でないか確認します。

### `already in call` と表示される

そのサーバーには既に通話状態があります。先に `/cw-leave` を実行してください。異常終了などでDiscord上の表示と状態が一致しない場合はBotプロセスを再起動します。

### `not in call` と表示される

先に `/cw-join` を実行してください。Botの再起動後はDiscord上にBotが一時的に残って見えても、内部の通話状態は失われています。

### 独自問題ファイルを認識しない

- 起動時のカレントディレクトリ直下に `lesson_txt/` があるか確認します。
- 指定したファイル名と大文字・小文字が一致しているか確認します。
- ファイル名に `/` が含まれていないか確認します。
